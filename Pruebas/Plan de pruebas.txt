OBJETIVO DEL PLAN DE PRUEBAS

El objetivo de este plan de pruebas es verificar la corrección funcional, la calidad de los datos, el rendimiento básico y la robustez del sistema desarrollado en el TFG, el cual implementa un pipeline completo para la construcción y explotación de un grafo de conocimiento a partir de fuentes de datos heterogéneas.

El sistema abarca desde la descarga de datos (scraping web, Wikidata y Kaggle), su normalización y unificación de entidades, hasta la creación de un grafo RDF en formato TTL y su consulta mediante una aplicación de escritorio que utiliza un servidor SPARQL local basado en Apache Jena Fuseki.

ALCANCE

Este plan cubre las siguientes fases:

Descarga de datos
Normalización de datos
Unificación de entidades
Creación del grafo RDF
Carga del grafo en servidor SPARQL
Aplicación de escritorio para consultas SPARQL
Pruebas unitarias
Pruebas básicas de rendimiento

Quedan fuera del alcance:

Pruebas de seguridad
Pruebas de escalabilidad a gran volumen
Pruebas en entornos distribuidos

FASE 1 – PRUEBAS DE DESCARGA DE DATOS

Objetivo:
Verificar que el script de scraping obtiene correctamente los datos desde la fuente web.

Casos de prueba:

Ejecución correcta del script con conexión a internet.
Generación del archivo CSV de salida.
Verificación de que el número de registros es mayor que cero.
Validación de columnas obligatorias.
Comprobación de valores nulos.
Manejo de errores HTTP (404, 500).
Comportamiento ante cambios menores en la estructura HTML.
Reejecución del script sin duplicar datos.

Resultado esperado:
Archivo CSV generado correctamente con datos completos y coherentes.

4.2 Script de descarga desde Wikidata

Objetivo:
